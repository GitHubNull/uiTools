## 简介
`uiTools` 是一个基于Java Swing的图形界面工具，用于处理字符串格式化、编解码操作以及XML/JSON数据的XPath和JSONPath表达式提取。本指南将指导您如何快速搭建和运行该项目，确保您能够顺利启动应用并开始使用其强大的功能。

## 环境要求
在开始之前，请确保您的开发环境满足以下要求：

- **JDK 11+**：项目需要Java 11或更高版本。您可以通过运行 `java -version` 命令来检查当前安装的Java版本。
- **Maven 3.6+**：项目使用Maven进行构建和依赖管理。您可以通过运行 `mvn -v` 命令来验证Maven的安装和版本。

**注意**：虽然CLAUDE.md文件中提到了JDK 17或更高版本，但根据pom.xml文件中的配置，项目实际要求的是JDK 11。因此，JDK 11及以上版本均可满足项目需求。

## 项目构建
按照以下步骤克隆仓库并构建项目：

### 1. 克隆仓库
首先，使用Git克隆项目仓库到本地：
```bash
git clone https://github.com/GitHubNull/uiTools.git
cd uiTools
```

### 2. 执行构建
使用Maven执行清理和打包操作：
```bash
mvn clean package
```
此命令会执行以下操作：
- `clean`：清理之前构建生成的文件
- `package`：编译源代码、运行测试（如果有），并将项目打包成可执行的JAR文件

构建成功后，您将在 `target/` 目录下找到生成的JAR文件，文件名类似于 `uiTools-1.1.8.jar`。

## 运行应用程序
构建完成后，您可以通过以下两种方式之一启动应用程序：

### 方式一：使用Maven运行
```bash
mvn exec:java -Dexec.mainClass="org.oxff.Main"
```
此命令会直接运行主类 `org.oxff.Main`，无需手动指定JAR文件。

### 方式二：运行打包后的JAR文件
```bash
java -jar target/uiTools-1.1.8.jar
```
此命令会启动打包好的JAR文件。请根据实际生成的JAR文件名调整命令中的版本号。

无论采用哪种方式，应用程序启动后将显示一个图形用户界面，您可以开始使用各种字符串处理功能。

## 常见问题排查
在初始化过程中，您可能会遇到一些常见问题。以下是排查提示：

### Java版本不匹配
如果启动时出现与Java版本相关的错误，请确保您的Java版本为11或更高。您可以通过以下命令检查：
```bash
java -version
```
如果版本过低，请升级到JDK 11或更高版本。

### Maven配置错误
如果Maven构建失败，请检查以下几点：
- 确保Maven已正确安装且版本为3.6或更高
- 检查网络连接，确保Maven可以下载所需的依赖项
- 如果构建过程中出现依赖项下载失败，可以尝试运行 `mvn clean package -DskipTests` 跳过测试以加快构建过程

### 程序无法启动
如果程序无法启动，请检查：
- 确认JAR文件是否完整且位于正确的目录下
- 查看控制台输出的错误日志，根据日志信息进行排查
