## 代码贡献

我们非常欢迎社区成员通过Pull Request贡献代码。以下是贡献代码的基本流程：

1. Fork项目仓库
2. 创建新的分支进行开发
3. 实现您的功能或修复
4. 确保代码符合项目风格规范
5. 添加必要的测试
6. 更新相关文档
7. 提交Pull Request

### 添加新操作

项目采用工厂模式管理各种操作，添加新操作非常简单：

1. **创建操作类**：实现`Operation`接口
   ```java
   public class YourOperation implements Operation {
       @Override
       public String execute(String input) {
           // 实现您的逻辑
           return processedResult;
       }

       @Override
       public OperationCategory getCategory() {
           return OperationCategory.YOUR_CATEGORY;
       }

       @Override
       public String getDisplayName() {
           return "您的操作名称";
       }
   }
   ```

2. **支持表达式功能（可选）**：添加重载的`execute`方法
   ```java
   public String execute(String input, String expressions) {
       // 处理表达式输入
       return extractedResults;
   }
   ```

3. **注册操作**：在`OperationFactory`中添加新操作到`allOperations`数组

## 开发环境设置

为了确保代码的一致性，请使用以下开发环境：

- **JDK 11** 或更高版本
- **Maven 3.6** 或更高版本

### 构建项目
```bash
git clone https://github.com/GitHubNull/uiTools.git
cd uiTools
mvn clean package
```

### 运行项目
```bash
# 使用Maven运行
mvn exec:java -Dexec.mainClass="org.oxff.Main"

# 或运行打包后的JAR文件
java -jar target/uiTools-1.1.4.jar
```

## 代码风格规范

为了保持代码库的一致性和可读性，请遵守以下代码风格规范：

- 使用UTF-8编码
- 采用4个空格缩进，不要使用Tab
- 遵循Java命名规范（驼峰命名法）
- 为所有公共方法和类添加JavaDoc注释
- 保持代码简洁，避免过度复杂的逻辑
- 使用有意义的变量和方法名
- 遵循单一职责原则

项目使用FlatLaf作为UI主题，请在UI相关的代码中保持一致的界面风格。

## 测试要求

我们重视代码质量，因此对贡献的代码有以下测试要求：

- 新功能必须包含相应的单元测试
- 修复的bug必须包含回归测试
- 测试代码应覆盖主要使用场景和边界条件
- 确保所有现有测试通过

虽然项目目前没有提供具体的测试框架配置，但建议使用JUnit等主流测试框架来保证代码质量。
